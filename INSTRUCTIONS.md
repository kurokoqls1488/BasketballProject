# Инструкция по настройке базы данных Supabase

## Проблема
При регистрации пользователя возникает ошибка:
```
AuthRetryableFetchException(message: {"code":"unexpected_failure","message":"Database error saving new user"}, statusCode: 500)
```

## Причина
В Supabase есть триггер, который автоматически пытается создать запись в таблице `public.users` при регистрации нового пользователя. Однако таблица `users` имеет поле `id` типа `integer`, в то время как Supabase Auth генерирует UUID. Это несоответствие типов вызывает ошибку при вставке.

## Решение

### Шаг 1: Удалите все существующие таблицы

1. Откройте Supabase Dashboard
2. Перейдите в раздел **SQL Editor**
3. Скопируйте содержимое файла [`drop_all_tables.sql`](drop_all_tables.sql)
4. Вставьте в SQL Editor и нажмите **Run**

### Шаг 2: Создайте новые таблицы

1. В том же **SQL Editor**
2. Скопируйте содержимое файла [`your_supabase_init.sql`](your_supabase_init.sql)
3. Вставьте в SQL Editor и нажмите **Run**

Этот скрипт создаст:
- Таблицу `users` с правильным типом поля `id` (UUID)
- Триггер для автоматического создания пользователя при регистрации
- Политики безопасности (RLS)
- Все остальные таблицы с правильными связями

### Шаг 3: Проверьте работу регистрации

После выполнения SQL-скрипта попробуйте зарегистрировать нового пользователя. Регистрация должна работать без ошибок.

## Что было изменено в коде

### lib/services/auth_service.dart

1. **Метод `registerUser`**: Убрана попытка ручного добавления пользователя в таблицу `users`, так как триггер делает это автоматически.

2. **Метод `loginUser`**: Исправлено получение данных пользователя из таблицы `users` (теперь `id` читается как `String`, а не преобразуется из `int`).

## Структура таблицы users

После выполнения SQL-скрипта таблица `users` будет иметь следующую структуру:

| Поле | Тип | Описание |
|------|-----|----------|
| id | UUID | Первичный ключ, ссылается на auth.users(id) |
| nickname | TEXT | Никнейм пользователя |
| email | TEXT | Email пользователя |
| id_role | INTEGER | Роль пользователя (по умолчанию 2) |
| avatar_url | TEXT | URL аватара |
| created_at | TIMESTAMP | Дата создания |
| updated_at | TIMESTAMP | Дата обновления |

## Триггеры

1. **on_auth_user_created**: Автоматически создает запись в таблице `users` при регистрации нового пользователя в Supabase Auth
2. **update_users_updated_at**: Автоматически обновляет поле `updated_at` при изменении данных пользователя

## Политики безопасности (RLS)

- Пользователи могут просматривать только свои данные
- Пользователи могут обновлять только свои данные
- Пользователи могут вставлять только свои данные
- Упражнения и тренировки доступны для просмотра всем
- Пользователи могут создавать, обновлять и удалять только свои упражнения и тренировки

## Важно

Данные упражнений, тренировок и избранного, которые были в старой базе данных, не будут перенесены автоматически, так как они содержали `id_user` типа `integer`, а теперь требуется UUID. После создания пользователей через Supabase Auth вы сможете добавить эти данные вручную или создать новые.
